# SWLRC v1 格式规范

## 1. 定位

SWLRC（Star Whisper Lyrics）是面向星语音乐盒与星语音库的增强时间轴歌词格式。

它用于表达：

- 行级歌词时间范围；
- 字级、词级或混合 token 时间范围；
- 全局歌词偏移；
- 少量可选歌词元信息。

SWLRC v1 以 UTF-8 纯文本保存，文件扩展名为：

```text
.swlrc
```

普通 `.lrc` 主要表达行级时间轴；`.swlrc` 在此基础上表达 token 级时间轴，用于逐字或逐词高亮。

## 2. 基本结构

一个 SWLRC 文件由三部分组成：

1. 头部元信息；
2. 空行；
3. 歌词块。

空行不承载语义，读取端应允许省略。

示例：

```swlrc
[swlrc:1]
[title:终于等到你]
[artist:张靓颖]
[duration:255000]
[offset:0]
[tokenization:char]

[01:22.450,01:23.650]
<01:22.450,01:22.680>终
<01:22.680,01:22.870>于
<01:22.870,01:23.120>等
<01:23.120,01:23.400>到
<01:23.400,01:23.650>你
[01:26.100,01:27.720]
<01:26.100,01:26.360>差
<01:26.360,01:26.620>点
<01:26.620,01:26.900>要
<01:26.900,01:27.180>错
<01:27.180,01:27.430>过
<01:27.430,01:27.720>你
```

## 3. 文件编码与换行

- 编码必须为 UTF-8。
- 推荐使用 LF 换行；CRLF 读取端应兼容。
- 空行不承载语义，可忽略。
- 注释在 v1 中暂不支持。

## 4. 头部元信息

元信息格式：

```text
[key:value]
```

### 4.1 必填字段

```swlrc
[swlrc:1]
```

说明：

- `swlrc`：格式标识；
- `1`：主版本号；
- 必须位于文件第一行；
- 不支持的主版本号必须拒绝解析或进入兼容降级流程。

### 4.2 推荐字段

```swlrc
[offset:0]
[tokenization:char]
```

#### offset

```swlrc
[offset:0]
```

全局时间偏移，单位为毫秒，可正可负。

生效规则：

```text
最终播放时间 = 文件内时间 + offset
```

示例：

```swlrc
[offset:350]
```

表示所有歌词整体延后 350ms 生效。

```swlrc
[offset:-250]
```

表示所有歌词整体提前 250ms 生效。

#### tokenization

```swlrc
[tokenization:char]
```

表示默认 token 切分方式。

v1 支持：

- `char`
- `word`
- `mixed`

含义：

- `char`：按字符或汉字高亮；
- `word`：按词高亮；
- `mixed`：允许一行中同时存在字符、词组、英文单词或人工定义片段。

未知 `tokenization` 是硬错误，读取端不得静默降级。

### 4.3 可选字段

```swlrc
[title:终于等到你]
[artist:张靓颖]
[album:我相信]
[duration:255000]
[language:zh-CN]
```

字段说明：

| 字段 | 含义 | 单位 |
| --- | --- | --- |
| `title` | 歌曲标题 | 无 |
| `artist` | 歌手名 | 无 |
| `album` | 专辑名 | 无 |
| `duration` | 音频总时长 | 毫秒 |
| `language` | 歌词主要语言 | BCP 47 风格语言标签 |

说明：

- 星语音库中的歌曲元数据是权威来源；
- SWLRC 中的 `title`、`artist`、`album` 仅用于导入校验、人工识别与文件脱离曲库后的可读性；
- `duration` 推荐填写，用于检测歌词和音频版本是否大概率匹配；
- 未知元字段允许存在，读取端应尽量保留；如果不支持保留，也可以忽略；
- 未知元字段不应导致整个文件无法读取。

## 5. 时间格式

SWLRC v1 使用绝对时间。

格式：

```text
MM:SS.mmm
```

示例：

```text
01:22.450
12:05.007
```

规则：

- 分钟可超过 59；
- 秒范围为 `00` 到 `59`；
- 毫秒固定为三位；
- 所有时间均相对于音频起点；
- 不使用相对时间或持续时长表达。

## 6. 歌词行

歌词行头格式：

```text
[startTime,endTime]
```

示例：

```swlrc
[01:22.450,01:23.650]
```

含义：

- 当前歌词行开始于 `01:22.450`；
- 当前歌词行结束于 `01:23.650`；
- 该歌词行之后的 token，直到下一歌词行头出现前，都属于当前歌词行。

每个歌词行必须至少包含一个 token。

## 7. Token

Token 行格式：

```text
<startTime,endTime>text
```

示例：

```swlrc
<01:22.450,01:22.680>终
```

一个 token 可以是：

- 单个汉字；
- 一个英文单词；
- 一个日文假名片段；
- 一个词组；
- 一个标点；
- 人工定义的任意可显示文本。

英文逐词示例：

```swlrc
<00:20.000,00:20.260>I
<00:20.260,00:20.700>will
<00:20.700,00:21.300>always
<00:21.300,00:21.680>love
<00:21.680,00:22.100>you
```

中文逐字示例：

```swlrc
<01:22.450,01:22.680>终
<01:22.680,01:22.870>于
<01:22.870,01:23.120>等
<01:23.120,01:23.400>到
<01:23.400,01:23.650>你
```

## 8. 时间与结构校验

解析器必须执行以下校验。

### 8.1 行级校验

```text
line.startTime < line.endTime
```

### 8.2 Token 范围校验

对于任意 token：

```text
line.startTime <= token.startTime < token.endTime <= line.endTime
```

### 8.3 Token 顺序校验

同一行内 token 必须按开始时间非递减排列：

```text
previousToken.startTime <= currentToken.startTime
```

推荐进一步要求 token 不重叠：

```text
previousToken.endTime <= currentToken.startTime
```

但 v1 读取端可容忍少量重叠，以兼容连读、和声或人工修正场景。

### 8.4 音频时长校验

若存在 `duration`：

```text
0 <= line.startTime < line.endTime <= duration
```

允许实现方提供小范围容错，例如 ±2000ms，用于处理音频版本时长差异。

### 8.5 语法错误、语义警告与容错项

语法错误应拒绝解析，例如：

- 缺失首行 `[swlrc:1]`；
- 时间格式非法；
- 歌词行或 token 语法非法；
- token 出现在任何歌词行之前；
- 缺失必需的 `[offset:...]` 或 `[tokenization:...]`；
- 未知 `tokenization`。

语义 warning 不一定拒绝文档，例如：

- 宽松模式下 token 重叠；
- 歌词时间轻微超出 `duration`。

以下输入细节属于容错项：

- 空行；
- CRLF 换行；
- 未知元字段。

## 9. 文本拼接规则

一行歌词显示文本由该行所有 token 的 `text` 按原始顺序直接拼接得到。

示例：

```swlrc
<01:22.450,01:22.680>终
<01:22.680,01:22.870>于
<01:22.870,01:23.120>等
<01:23.120,01:23.400>到
<01:23.400,01:23.650>你
```

显示文本为：

```text
终于等到你
```

SWLRC v1 不单独保存行级 `text` 字段，避免与 token 文本产生双份数据和一致性问题。

## 10. 播放高亮规则

在给定播放位置 `playbackMs` 时：

```text
effectivePlaybackMs = playbackMs - offset
```

当前 token 的匹配条件：

```text
token.startTime <= effectivePlaybackMs < token.endTime
```

当前歌词行的匹配条件：

```text
line.startTime <= effectivePlaybackMs < line.endTime
```

建议渲染行为：

- 当前 token：高亮；
- 已完成 token：已唱状态；
- 未到达 token：未唱状态；
- 当前行：可进行放大、居中或强调显示；
- 行结束到下一行开始之间：保留上一行或进入过渡状态，由客户端 UI 决定。

## 11. v1 明确不包含的能力

以下能力不属于 SWLRC v1：

- 翻译歌词；
- 音译歌词；
- 多人演唱角色；
- 和声轨；
- 歌词颜色或动画样式；
- K 歌评分；
- 音高曲线；
- 编辑历史；
- 对齐器置信度；
- 人工审核状态；
- 签名、加密或 DRM。

这些内容应由星语音库数据库、对齐器中间产物或未来版本承载。

## 12. 兼容策略

- `.lrc`：作为行级歌词兼容格式；
- `.swlrc`：作为逐字 / 逐词歌词增强格式；
- 音乐盒优先加载 `.swlrc`；
- 若无 `.swlrc`，回退加载 `.lrc`；
- 若无 `.lrc`，可展示纯文本歌词或无歌词状态。

推荐优先级：

```text
SWLRC
-> LRC
-> 纯文本歌词
-> 无歌词
```

## 13. 最小合法示例

```swlrc
[swlrc:1]
[offset:0]
[tokenization:char]
[00:10.000,00:11.000]
<00:10.000,00:10.300>你
<00:10.300,00:10.600>好
<00:10.600,00:11.000>吗
```

## 14. 设计原则

SWLRC v1 的目标是：

- 人类可读；
- 程序易解析；
- 时间轴可校验；
- 对齐器易生成；
- 人工易修正；
- 音乐盒易渲染；
- 可与普通 LRC 并存；
- 为未来扩展保留版本边界。

SWLRC 不负责描述 UI 效果，只负责描述“唱什么，以及何时唱”。
